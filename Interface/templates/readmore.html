User Guide

This user manual will guide you to how to go through the process. This manual has been written to help you to understand the differences between “Normal Mode” and “Professional Mode”, and how to fill in information.

1. Click "Get Started"

As shown in Figure 1, click this button and turn to mode selection page.

Figure 1

2.Choose mode

In Figure 2, we can see two kinds of modes can be choice: “Normal Mode” and “Professional Mode”. As their name implies, “Normal Mode” is the mode which is suitable to every user whoever a new hand or old hand.

Normal Mode: If it is your first try, we recommend you to choose it. “Normal Mode” has an easier operation procedure and detailed guidance. Click button “TRY IT” and move user manual to part “3. Normal Mode”.

Professional Mode: If you an old hand, you can try the “Professional Mode”. This mode has more detail can input and harder to operate. Meanwhile, you can get a more precise result. Click button “GO!” and move user manual to part “4. Professional Mode”.

Figure 2

3.Normal Mode


3.1.Base information input

This part is the most base and important information of house. It directly affects the prediction result. Please fill in as much as possible.

3.1.1 Location information

As shown in the Figure 3.1.1, we need to input “address”, “zipcode”, “latitude” and “longitude”. This part is optional, you can choose what you know to fill in. Filling in can improve accuracy of search.

  • Address: The address of your house.
  • Zipcode: Input the zipcode of the house can improve accuracy of the search. It is helpful to narrow down the search. E.p. 98001.
  • Latitude: Input the latitude of the house can improve accuracy of the search. It is helpful to narrow down the search. E.p. 47.
  • Longitude: Input the longitude of the house can improve accuracy of the search. It is helpful to narrow down the search. E.p. -122.

Notes:

  1. You may need to use some mapping software, such as Google Maps, to get a better idea of the latitude and longitude of the place.
  2. Since the software's datasets is mainly concentrated in a small town in the United States, in order to make the data more accurate, the software currently supports the query of house price estimates in that place.
Figure 3.1.1

3.1.2 General information

As shown in the Figure 3.1.2, we need to input “Bedrooms number”, “Bathrooms number”, “Living size” and “Lot size”.

  • Bedrooms number: The number of bedrooms in the house. E.p. 2.
  • Bathrooms number: The number of bathrooms in the house. E.p. 2.
  • Living size: This is often used to describe the internal living area of a house, excluding non-residential spaces such as garages, basements, etc. E.p. 1000.

Notes:

The unit we use here is square footage.1 square meter is equal to 10.7639 square feet.

Figure 3.1.2

3.1.3 Details information

As shown in the Figure 3.1.3, we need to input “Floors”, “Is waterfront?”, “The size above basement” and “Basement size”.

  • Floors: The number of floors. Don’t need to mind the basement. E.p. 2.
  • Is waterfront?: Whether your house is near the water. Usually, the price of a house by the water is higher. E.p. Yes.
  • The size above basement: The total area of the house above ground level. It includes the attic. E.p. 1400.
  • Basement size: The total area of basement. E.p. 60. If this house don’t have the basement, please fill 0 instead.

Notes:

The unit we use here is square footage.1 square meter is equal to 10.7639 square feet.

Figure 3.1.3

3.1.4 More information

As shown in the Figure 3.1.4, we need to input “Built year”, “Renovated year(if not, enter 0)”, “Living size in 2015” and “Lot size in 2015”. This part is optional, you can choose what you know to fill in. Filling in can improve accuracy of prediction result.

  • Built year: The year of house when was built. E.p. 2000.
  • Renovated year(if not, enter 0): The year of house when was repaired. If the house was never repaired, enter “0”. E.p. 0.
  • Living size in 2015: The size of living space in 2015. Usually, it is same as now. E.p. 100.
  • Lot size in 2015: The total area of house in 2015. Usually, it is same as now. E.p. 120.

4.2 Single Prediction

As figure 4.2.1, we need to enter many parameters. These parameters can be divided into 4 parts: “Location”, “Basic”, “Advance” and “Pro settings”.

Location:

  • Zipcode:Input the zipcode of the house can improve accuracy of the search. It is helpful to narrow down the search. E.p. 98001.
  • Lat: Input the latitude of the house can improve accuracy of the search. It is helpful to narrow down the search. E.p. -47.
  • Long: Input the longitude of the house can improve accuracy of the search. It is helpful to narrow down the search. E.p. 120.
  • Is waterfront?: Whether your house is near the water. Usually, the price of a house by the water is higher. 1 is for yes, 0 is for no. E.p. 1.

Notes:

  1. You may need to use some mapping software, such as Google Maps, to get a better idea of the latitude and longitude of the place.
  2. Since the software's datasets is mainly concentrated in a small town in the United States, in order to make the data more accurate, the software currently supports the query of house price estimates in that place.


Basic:

  • Bedrooms number: The number of bedrooms in the house. E.p. 2.
  • Bathrooms number: The number of bathrooms in the house. E.p. 2.
  • Floors: The number of floors. Don’t need to mind the basement. E.p. 2.
  • Built year: The year of house when was built. E.p. 2000.
  • Renovated year(if not, enter 0): The year of house when was repaired. If the house was never repaired, enter “0”. E.p. 0.
  • Living size: This is often used to describe the internal living area of a house, excluding non-residential spaces such as garages, basements, etc. E.p. 1000.
  • Lot space: The total area of house. It includes exterior space such as garden. E.p. 140.
  • Above basement space (in ft square): The total area of the house above ground level. It includes the attic. E.p. 120.
  • Basement size: The total area of basement. If this house don’t have the basement, please fill 0 instead. E.p. 20.

Notes:

The unit we use here is square footage.1 square meter is equal to 10.7639 square feet.



Advance:

  • View number: You need to input the times this property has been viewed already. If you don’t sure, you can enter 0. E.p. 2.
  • Condition: You need to score for the condition. E.p. 3.
  • Grade: You need to evaluate and give a grade. If you don’t sure, you can enter 0. Usually, the grade not over 15. E.p. 4.
  • Living space in 2015 (in ft square): The size of living space in 2015. Usually, it is same as now. E.p. 120.
  • Lot space in 2015 (in ft square): The total area of house in 2015. Usually, it is same as now. E.p. 140.

Notes:

The unit we use here is square footage.1 square meter is equal to 10.7639 square feet.



Pro setting:

  • Enable LLM: Choose to use LLM algorithm to generate prediction price.
  • Enable Full Prediction: Choose to do full prediction to generate prediction price.
  • Using user-defined confidence levels: Choose this option. You can decide the confidence levels by yourselves. If you choose this option, you need to enter the number of confidence levels.
  • Confidence levels: The number of confidence levels. Usually, the larger the number, the more accurate the result.
  • Enable Hidden Prediction: Choose to use hidden prediction.
  • Select Model Core: We can choose “Random Forest” or “XGBoosat”.

4.3 Batch Prediction

This page, you can do batch prediction by uploading file in proper format. As figure 4.3.1, this page can be divided into 2 parts: “Pro settings” (this part is to choose the kind of model.) and “Select data file” (this part is to upload file).

Pro setting:

  • Enable LLM: Choose to use LLM algorithm to generate prediction price.
  • Enable Full Prediction: Choose to do full prediction to generate prediction price.
  • Using user-defined confidence levels: Choose this option. You can decide the confidence levels by yourselves. If you choose this option, you need to enter the number of confidence levels.
  • Confidence levels: The number of confidence levels. Usually, the larger the number, the more accurate the result.
  • Enable Hidden Prediction: Choose to use hidden prediction.
  • Select Model Core: We can choose “Random Forest” or “XGBoosat”.

Select data file:

Click the “BROWSE” to select the data file. And you can see it on the left after chose.After doing these things, please click the “SUBMIT”. And you will get the result if you upload file with right format.

Notes:

the true format is .csv format.
